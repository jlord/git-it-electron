Crie um arquivo em seu novo repositório, adicione algo a este arquivo e salve as mudanças usando Git.

Commits

Commits (cometimentos, "envios") são o núcleo do uso de Git. São os momentos em que você salva e descreve o que você fez. Pense neles como instantes na linha do tempo da história do seu projeto.

Criando um Novo Arquivo

Agora que você tem um novo repositório inicializado, vamos adicionar um arquivo a ele.

Abra seu editor de texto e crie um arquivo vazio. Agora, escreva um pequeno texto, talvez um simples "Olá!", e salve o arquivo como "readme.txt" na pasta "hello-world" que você criou no último desafio.

Estado, Adicionar e Cometer Mudanças

A seguir, cheque o estado (status) do seu repositório para confirmar se houveram mudanças. Você sabe que algo mudou, mas será que o Git sabe disso?

Certifique-se de que você está na sua pasta hello-world enquanto você está executando estes comandos. Use o Git para verificar mudanças no seu repositório:

Primeiramente, cheque o estado:

git status

O Git deverá lhe dizer que um arquivo foi adicionado.

Então, adicione o arquivo que você acabou de criar para que ele se torne parte das mudanças que você fará no commit (em outras palavras, salvar) com o Git:

git add readme.txt

Finalmente, faça o commit destas mudanças para o histórico do repositório com uma curta mensagem (m) descrevendo as alterações.

git commit -m "Created readme"

Passo: Faça Mais Mudanças

Agora, adicione outra linha em 'readme.txt' e salve o arquivo novamente.

No terminal, você pode ver a differença entre o arquivo atual e como ele estava durante o seu último commit.

Diga ao Git que você quer ver as diferenças:

git diff

Agora, com o conhecimento que você acabou de obter, faça o commit das mais recentes mudanças.
