Repositórios

Um repositório é uma coleção de itens relacionados. No nosso caso, quando escrevemos softwares, é uma coleção de arquivos relacionados a projetos dos softwares. Você pode imagina-lá como uma pasta de projeto com todos os arquivos relevantes dentro dela. Na verdade, isso é o que vai parecer no seu computador de qualquer maneira. Algumas vezes eles são chamados de "repos" para abreviar.

Você diz ao Git o que seu projeto é e o Git vai começar a monitorar todas as mudanças dessa pasta. Isto torna-o um repositório Git - uma pasta de itens que estão sendo monitoradas pelo Git. O Git monitora quando arquivos são adicionados, removidos ou até mesmo se uma única letra em um único arquivo é alterada. Tudo isso mais quem o fez e quando fez é monitorado pelo Git. Nos software isso é chamado controle de versão.

Usando o Terminal

O Terminal (ou Bash) é uma maneira de instruir seu computador a fazer coisas digitando comandos em vez de clicar com seu mouse. Você pode renomear arquivos, abrir arquivos, criar novas pastas, mover-se entre diretórios (pastas) e mais tudo executando digitando comandos. Você pode até mesmo executar um editor de texto (tal como o Vim) no seu terminal e nunca ter que sair dele!

Além de navegar no seu computador, você pode também usar programas no Terminal que tem uma command-line interface (CLI), que significa que eles podem ser exexutados com comandos no terminal. Git é um desses. Você pode acessar e controlar Git através de comandos no terminal, como você vai estar fazendo muito em breve!

Em Git-it, você vai aprender algumas ações basicas de linhas de comando (além dos comandos Git) o qual serão descritos nos passos a seguir.

Criar um repositório

Você vai criar uma nova pasta do projeto e então inicializa-a como um repositório Git. Para facilitar as coisas, nomeie sua pasta com o nome do projeto, como 'hello-world'.

Abra seu terminal! Na sua janela de terminal, digite esses comandos, um por vez, pressione enter/return após cada comando.

Primeiro, crie uma nova pasta:
Dica: mkdir significa criar diretório

mkdir hello-world

Em seguida ir dentro dessa pasta:
Dica: cd significa mudar diretório

cd hello-world

Finalmente, diga ao Git para iniciar (começar a monitorar) esta pasta:

git init

O último comando deve retornar algo começando com "Initialized empty Git repository" (Inicializado repositório vazio do Git). Os outros comandos não retornam nada.

Você terminou! Se você quiser ter duplamente certeza que é um repositório Git, digite git status e se não retornar 'fatal: Not a git repository...' (Não é um repositório git), está tudo certo!
