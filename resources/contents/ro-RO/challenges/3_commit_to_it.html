Creează un fișier în noul depozit, adaugă ceva la acel fișier și operează acea schimbare cu Git.

Operări

Operările sunt esența folosirii Git. Sunt momentele în care salvezi și descrii ceea ce ai lucrat. Sunt bifele din desfășurătorul istoriei proiectului tău.

Creează un nou fișier

Acum că ai început un depozit, haide să adaugăm fișiere la el.

Deschide editorul de text și creează un nou fișier gol. Acum scrie un mic text, poate "Hello!", și salvează fișierul ca 'readme.txt' în directorul 'hello-world' pe care l-ai creat în ultima provocare.

Verifică starea + adaugă și operează schimbări

În continuare verifică starea depozitului pentru a afla dacă au existat schimbări. Știi că ai schimbat ceva, dar știe și Git?

Asigură-te că ești încă în directorul 'hello-world' când execuți aceste comenzi. Folosește Git pentru a vedea ce s-a schimbat în depozitul tău:

Întâi, verifică starea:

git status

Git ar trebui să îți spună că a fost adăugat un fișier.

Apoi adaugă fișierul pe care tocmai l-ai creat astfel încât să devină parte din schimbările pe care le vei opera (salva) cu Git:

git add readme.txt

În sfârșit, operează acele schimbări în istoria depozitului cu un scurt (m) mesaj care descrie schimbările.

git commit -m "Creat readme"

Pas: Fă mai multe schimbări

Acum adaugă o altă linie la 'readme.txt' și salvează fișierul din nou.

În terminal, poți vizualiza diferența dintre fișierul de acum și cum era la ultima operare.

Spune-i lui Git să arate diferența:

git diff

Acum, cu ce ai învățat mai sus, operează această ultimă schimbare.
