Adaugă o ramură, local, depozitului tău bifurcat pentru a lucra la schimbări.

Ramuri

Depozitele Git folosesc ramuri pentru a izola contribuțiile când este nevoie. Este o practică obișnuită când lucrezi la un proiect sau cu alții la un proiect să creezi o ramură pentru a păstra schimbările în lucru. În acest fel poți lucra în timp ce ramura principală, numită obișnuit 'master', rămâne stabilă. Când ai terminat de lucrat pe ramura ta, o fuzionezi înapoi în ramura 'master'.

Diagrama de mai jos prezintă cum poți ramifica de pe ramura 'master', lucra și apoi fuziona acele schimbări înapoi în 'master'. Poți chiar să ramifici și dintr-o ramură, ramura 'master' nu trebuie să fie baza.

Pentru o bună vizualizare a modului în care funcționează ramurile într-un proiect, vezi acest ghid GitHub: guides.github.com/overviews/flow

Pagini GitHub

GitHub va servi automat și va găzdui fișiere web statice în ramuri numite 'gh-pages'. Acest serviciu gratuit se numește GitHub Pages (Pagini GitHub). Deoarece proiectul pe care l-ai bifurcat creează un site, ramura sa principală se numește 'gh-pages' în loc de 'master'. Toate depozitele care au o ramură 'gh-pages' cu fișiere web se găsesc, active online, folosind acest tipar pentru URL:

http://githubusername.github.io/repositoryname

Creează o ramură

Când creezi o ramură, Git copiază totul din ramura curentă în care te afli și îl pune în ramura pe care ai cerut-o să fie creată.

Cât încă te afli în depozitul local 'patchwork', tastează git status pentru a vedea în care ramură te afli. Git ar trebui să răspundă că ești pe ramura 'gh-pages'.

Acum creează o nouă ramură și numește-o „add-<numeutilizator>”, unde 'numeutilizator' este numele tău de utilizator. De exemplu, "add-jlord". Ramurile diferențiază litere mari, așa că numește-ți ramura exact cum apare numele tău de GitHub.

git branch <NUMERAMURĂ>

Acum ai o ramură cu un nume noi care este identică cu 'gh-pages'.

Pentru a trece pe acea ramura și a lucra la ea schimbă ramură. Du-te pe noua ta ramură:

git checkout <NUMERAMURĂ>

Pas: Creează un nou fișier

În editorul de text:

  • Creează un nou fișier numit "add-<NUMEUTILIZATOR>.txt", unde 'numeutilizator' este numele tău de utiliator. De exemplu, "add-jlord.txt".
  • Apoi, scrie-ți numele de utilizator de GitHub în el, doar atât. De exemplu, eu aș tasta 'jlord'.
  • Salvează acest fișier în folderul 'contributors' din Patchwork: Patchwork/contributors/add-yourusername.txt
  • În continuare, operează-ți schimbările (vezi mai jos).

Operează

Urmează pașii pentru a opera într-un proiect:

git status

git add <contributors/NUMEFIȘIER>

git commit -m "mesaj de operare"

Acum împinge-ți schimbărea pe bifurcația ta, 'origin', pe GitHub:

git push origin <NUMERAMURĂ>

Permission denied...error: 403

Împingi schimbări într-un depozit unde nu ai acces de scriere. În acest caz, probabil împingi în 'jlord/patchwork' originar. Asigură-te că împingi în 'origin' și că indică spre adresa bifurcației tale de pe GitHub. Pentru a verifica și a vedea care sunt depozitele la distanță și unde indică, execută git remote -v. Ar trebui să ai 'upstream' indicând spre 'jlord/patchwork' și 'origin' indicând spre 'numeletăudeutilizator/patchwork'.

Pentru a ajusta un depozit la distanță care indică spre locul greșit, îi poți reseta url-ul: git remote set-url origin URLCĂTREBIFURCAȚIE .

Authentication failed...error: 401

Identitatea nu a putut fi verificată. S-ar putea să ai activată 2FA (Autentificare în doi factori) caz în care trebuie să folosești un token personal de acces ca parolă. Poți genera unul urmând aceste instrucțiuni. Va trebui să păstrezi asta și să o folosești în locul parolei când îți este solicitată. Poți de asemenea să salvezi acest token de acces pe calculatorul tău ca să nu fie nevoie să îl retastezi.

Fișierul NU este în folderul contributors

Fișierul pe care îl creezi ar trebui să fie plasat în interiorul folderului 'contributors' existent din depozitul Patchwork. Dacă l-ai pus altundeva, pur și simplu folosește Finder sau Windows Explorer pentru a-ți muta fișierul în folder. Poți verifica starea GIT iar și iar și vei descoperi că îți vede schimbările. Indexează și apoi operează toate (-A) aceste schimbări (adăugări și ștergeri) cu comenzile de mai jos.

git add -A

git commit -m "mută fișierul în folderul contributors"

Branch name expected: _____

Numele ramurii ar trebui să reflecte numele exact. Pentru a schimba numele ramurii:

git branch -m <NUMENOUDERAMURĂ>

Când ai făcut modificările, verifică din nou!

  • Poți crea și schimba pe o ramură cu un singur rând
  • git checkout -b <NUMERAMURĂ>
  • Creează o ramură nouă
  • git branch <NUMERAMURĂ>
  • Treci pe o ramură
  • git checkout <NUMERAMURĂ>
  • Listează ramurile
  • git branch
  • Redenumește o ramură pe care te afli
  • git branch -m <NUMENOUDERAMURĂ>
  • Verifică pe ce ramură lucrezi
  • git status
