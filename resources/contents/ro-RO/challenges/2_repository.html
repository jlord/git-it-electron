Depozite

Un depozit este o colecție de elemente înrudite. În cazul nostru, când scriem soft, este o colecție de fișiere legate de un proiect soft. Ți-l poți imagina ca pe un folder de proiect cu toate fișierele relevante în interiorul său. De fapt, așa va arăta pe calculatorul tău oricum. Uneori sunt numite "depouri" pe scurt.

Îi spui lui Git ce este proiectul tău și Git va începe să urmărească toate schimbările din acel folder. Această îl face depozit Git: un folder cu elemente care sunt urmărite de Git. Git urmărește când sunt adăugate fișiere, șterse sau chiar dacă o singură literă dintr-un singur fișier se schimbă. Toate acestea plus cine a făcut-o când este urmărit de Git. În soft, urmărirea schimbărilor în acest fel se numește controlul versiunii.

Folosirea terminalului

Terminalul (sau Bash) este un mod de a-ți instrui calculatorul să facă lucruri tastând comenzi în loc de a face click pe aplicații cu mouse-ul. Poți redenumi fișiere, deschide fișiere, crea noi foldere, naviga între directoare (foldere) și mai multe, toate executând comenzi tastate. Poți chiar folosi un editor de text pentru cod (ca Vim) în terminal și să nu fie nevoie să pleci niciodată!

Pe lângă navigarea prin calculator, poți de asemenea folosi programe în terminal care au interfețe în linie de comandă (CLI), însemnând că pot fi executate cu comenzi din terminal. Git este unul dintre acestea. Prima parte a comenzii îi spune calculatorului tău că vorbești cu Git. Părțile după aceea sunt comenzile și diferitele opțiuni pe baza cărora vrei să acționeze Git.

În Git-it vei învăța câteva acțiuni de bază în linia de comandă (pe lângă comenzile Git) pentru a naviga în calculator; sunt descrise în pași.

Creează un depozit

Vei crea un nou folder de proiect și apoi îl vei inițializa ca depozit Git. Vom numi folderul 'hello-world' în această provocare, dar poți alege un nume diferit dacă ai deja un folder numit așa (este un nume comun pentru un prim proiect).

E momentul să îți deschizi terminalul!

În fereastra terminalului, tastează aceste comenzi, pe rând, apăsând ENTER după fiecare.

Mai întâi, creeaza un nou folder:
Sugestie: mkdir înseamnă make directory (creează director)

mkdir hello-world

Apoi du-te în acel folder:
Sugestie: cd înseamnă change directory (schimbă director)

cd hello-world

La final, spune-i lui Git să inițializeze (să înceapă urmărească) folderul în care te afli acum:

git init

Ultima comandă ar trebui să întoarcă ceva care începe cu "Initialized empty Git repository". Celelalte comenzi nu întorc nimic.

Ai reușit! Dacă vrei să te asiguri că este un depozit Git, tastează git status și dacă nu întoarce 'fatal: Not a git repository...' este perfect!
