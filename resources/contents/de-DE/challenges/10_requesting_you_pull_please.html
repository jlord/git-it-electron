Pull-Request

Wenn Du Änderungen oder Verbesserungen an einem abgespalteten Projekt gemacht hast, möchtest du diese Änderungen meist auch an das originale Projekt senden, damit alle davon profitieren können. Dazu fragst du beim Projekt an (engl. request), dass deine Änderungen in das Original-Repository gezogen werden (engl. pull) - Diese Anfrage ist ein Pull-Request.

Wir möchten dich nun zur Liste der fertigen Workshop-Teilnehmer hinzufügen. Erstelle also ein Pull-Request, um deine Nutzernamen-Datei zum originalen Repository (github.com/jlord/patchwork) hinzuzufügen.

Erstelle ein Pull-Request

Öffne die Webseite des originalen Repository auf Github, von dem du dein Projekt abgespalten hast. In unserem Fall ist dies http://github.com/jlord/patchwork.

Oft erkennt GitHub bereits, wenn du einen neuen Branch in dein abgespaltetes Repository geschoben hast und zeigt dir dies im oberen Teil der Webseite des originalen Repository an. Wenn du dies mit deinem Branch 'add-<NutzerName>' siehst, kannst du dort direkt ein Pull-Request zu erzeugen. Falls nicht:

  • Klicke auf den grünen Button 'New pull request' (dt. neuer Pull-Request).
  • Wähle den Branch mit den Änderungen aus, die du hinzufügen willst. In unserem Fall sollte dies der Branch 'add-<NutzerName>' sein.

Du siehst nun eine Seite mit den Details des Pull-Request, den du erstellen möchtest. Die Seite zeigt dir die Commits und die jeweiligen Änderungen an - in Form eines Vergleiches zum ausgewählten 'gh-pages'-Branch des Original-Repositorys.

Wenn das Original-Repository eine Beitragsdokumentation (engl. contribution documentation) besitzt, wird GitHub darauf verweißen. Das ist die Dokumentation des Repository-Besitzers, wie zum Projekt beigetragen werden kann. Oft ist es sehr nützlich diese (im Vorraus) zu lesen, damit Änderungen akzeptiert werden!

Wenn auf dieser Seite alles gut - und so, wie erwartet - aussieht:

  • Klicke auf 'Create pull request' (dt. Pull-Request erstellen)
  • Füge einen Titel und eine Beschreibung der Änderungen hinzu, die du dem Autor des orignalen Repository vorschlägst.
  • Klicke auf 'Send pull request' (dt. Pull-Request absenden)!

Bingo! Du hast ein Pull-Request eingereicht. - Nimm dir ein paar Sekunden Zeit, um den Moment zu genießen.

Wenn mit deinem Pull-Request alles in Ordnung ist, sollte er in Kürze akzeptiert werden. Falls er nicht innerhalb von ein paar Minuten akzeptiert wird, hast du wahrscheinlich einen Kommentar von @reporobot bekommen, mit der Info, warum er die Änderungen nicht akzeptieren konnte. Sollte dies der Fall sein, schließe dein Pull-Request auf GitHub, ändere die nötigen Dinge in deinem Branch, schiebe diese Änderungen in dein abgespaltenes Repository und öffne deinen Pull-Request dann erneut. @reporobot wird es sich dann nocheinmal anschauen.
