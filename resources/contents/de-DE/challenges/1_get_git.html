Git

Git ist eine kostenfreie Open-Source Software (der Quellcode ist frei verfügbar), geschrieben von Linus Torvalds, dem Entwickler des Linux-Kernels.

Git ist eine Software, die Datei-Änderungen über den Verlauf der Zeit nachverfolgt, was in der Softwareentwicklung auch als Versionsverwaltung bezeichnet wird. Wenn Du bereits eine Funktion wie z.B. "Änderungen nachverfolgen" in Microsoft Word verwendet hast, ist dir dieses Konzept bereits bekannt!

Git installieren

Wir empfehlen Dir, Git mit der GitHub Desktop App auf deinem Computer zu installieren. Zwar werden wir die Desktop-App in Git-it nicht aktiv verwenden (wir lernen Git als Terminal-Anwendung!), sie enthält jedoch die Installation von Git und ist somit der einfachste Weg Git plattformunabhängig zu installieren.

  • Windows: Benutze die Git Shell als Dein Terminal.
  • Mac: Öffne GitHub Desktop, um in den Einstellungen die Kommandozeilen-Tools zu installieren. Benutze die Terminal-App als Dein Terminal.

Du hast Git schon installiert oder bist dir nicht sicher? Tippe git --version in dein Terminal ein, wenn Du in der Ausgabe eine Versions-Nummer größer als 1.7.10 siehst, ist die passende Software installiert. Weitere Informationen dazu findest Du auf der Git Webseite.

Git Software

Die GitHub Desktop App stellt zwar einen Großteil, jedoch leider nicht alle Funktionen von Git dem Nutzer zur Verfügung. Daher ist es wichtig, auch die Terminal-Funktionalität von Git zu kennen. Sobald Du diese jedoch beherrschst, wird Dir die Desktop App eine nette Hilfe sein, da sie (wie die GitHub Webseite) Deine Projekte übersichtlich darstellt.

Git selbst ist kein typisches Programm auf deinem Computer. Du wirst zwar kein Desktop-Icon sehen, aber trotzdem wird Git immer für Dich verfügbar sein und Du wirst es aus dem Terminal oder durch Git Desktop-Anwendungen nutzen können.

Git konfigurieren

Sobald Du GitHub Desktop (und somit Git) installiert hast kannst du Dein Terminal öffnen. Du kannst die Installation von Git mit folgendem Befehl überprüfen:

git --version

Dies wird dir die Versions-Nummer deiner Git-Installation zurückgeben und sollte in etwa so aussehen:

git version 1.9.1

Als nächstes kannst du Git so konfigurieren, dass es Deine Arbeit mit Dir verknüpfen kann:

Speichere Deinen Namen:

git config --global user.name "Dein Name"

Speichere deine E-Mail:

git config --global user.email "DeineEmail@Beispiel.de"

Damit hast Du die erste Aufgabe erledigt! Klicke auf den 'Prüfen'-Button, um deine Arbeit zu kontrollieren.

Dollar-Zeichen in der Code-Dokumentation

Dollar-Zeichen $ werden oft in Programm-Dokumentationen verwendet um zu signalisieren, dass diese Zeile Kommandozeilen-Code enthält (vergleiche die Code-Schnipsel oben). Das $ wird dann aber nicht in die Kommandozeile übernommen, sondern nur der darauf folgende Text wird verwendet.
