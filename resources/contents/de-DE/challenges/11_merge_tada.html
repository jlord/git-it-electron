Aufräumen

Wenn du Branches nicht mehr benötigst, kannst du sie sowohl lokal, als auch auf dem Remote löschen. Dies hilft, um nicht irgendwann mit einem Haufen alter Branches dazustehen.

Führe lokal zusammen

Dein Pull-Request wurde akzeptiert! Dein Branch wurde in den Branch 'gh-pages' des originalen GitHub-Repository integriert (engl. merge). Du kannst nun Deinen lokalen Branch auch integrieren.

Zuerst, wähle den Branch aus, in den Du hinein integrieren möchtest. In unserem Fall der Branch 'gh-pages'.

git checkout gh-pages

Sage nun Git, welchen Branch du integrieren möchtest, in diesem Fall Deinen Branch, der mit "add-" anfängt.

git merge <BRANCHNAME>

Räume auf, indem Du Deinen Branch löschst. Da die Änderungen bereits zusammengeführt wurden, brauchst Du den Branch nicht mehr wirklich.

git branch -d <BRANCHNAME>

Du kannst den Branch auch von Deinem Remote-Repository auf GitHub löschen:

git push <REMOTENAME> --delete <BRANCHNAME>

Vom Upstream ziehen

Letztlich hat sich auch das Original verändert, zumal Dein Pull-Request akzeptiert wurde. - Reporobot hat Deinen Namen zur Webseite hinzugefügt! Wenn Du diese Änderungen nun vom Upstream ziehst, wird Dein Repository wieder aktuell sein und die Version mit Deinem Namen zeigen. Du kannst die Webseite deines Repository live unter der folgenden Url anschauen: DeinNutzerName.github.io/patchwork.

Um vom originalen Repository 'upstream' zu ziehen:

git pull upstream gh-pages
