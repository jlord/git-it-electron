Commits

Commits ('to commit' - dt. etw. begehen/übergeben/überlassen) sind weitere Kernbestandteile der Git-Struktur. Sie bezeichnen die Momentaufnahmen, die gespeichert werden und beschreiben somit die Veränderungen, die gemacht wurden. Commits sind die markierten Zeitpunkte auf dem Zeitstrahl der Projekt-Historie.

Erzeuge eine neue Datei

Nun, da Du ein Repository eröffnet hast können wir damit beginnen eine Datei hinzuzufügen.

Öffne Deinen Texteditor in einer neuen leeren Datei. Schreibe ein bisschen text (z.B. "Hallo!") und speichere die Datei als "readme.txt" in den "hello-world" Ordner, den du in der letzten Aufgabe erstellt hast.

Überprüfe den Status und übergib die Änderungen

Als nächstes überprüfe den Status deines Repositorys, um herauszufinden, ob dort etwas geändert wurde. Du weißt natürlich, dass Du etwas geändert hast, aber weiß Git das auch?

Stelle sicher, dass Du noch immer in Deinem "hello-world" Verzeichnis arbeitest, wenn Du die folgenden Kommandos ausführst. Benutze Git um zu sehen, was sich in deinem Repository verändert hat:

Zuerst, überprüfe den Status:

git status

Git sollte dir nun sagen, dass eine Datei hinzugefügt wurde.

Als nächstes, füge die Datei hinzu (engl. add), damit es Teil der Änderungen wird, die du dann mit Git sichern bzw. übergeben (commit) wirst:

git add readme.txt

Nun übergib (commit) diese Änderungen and die Historie deines Repository mit einer kurzen Nachricht (engl. message - m, die deine Änderungen beschreibt.

git commit -m "readme erzeugt"

Nächster Schritt: Verändere etwas

Füge nun deiner Datei 'readme.txt' eine weitere Zeile hinzu und speichere die Änderungen.

Im Terminal kannst Du dir nun den Unterschied (engl. difference - diff) zwischen der aktuellen Datei und der Datei bei Deinem letzten Commit anzeigen lassen.

Lass Dir den Unterschied (diff) anzeigen:

git diff

Übergib nun Deine letzen Änderungen so wie Du es zuvor gelernt hast.
