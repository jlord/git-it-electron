Spalte auf GitHub ein Projekt ab und klone es auf deinen Computer.

Forks - Abspaltungen

Bisher hast du ein Projekt lokal neu erstellt und es dann zu GitHub geschoben. Aber das ist nur der halbe Spaß! Die andere Hälfte ist an fremden Projekten mitarbeiten zu können!

Wenn du ein Repository abspaltest (engl. fork), erzeugst du eine aktuelle Kopie des Projektes in deinem eigenen GitHub Account. Die abgespaltete Kopie beginnt somit ihre Existenz als ein eigenes Remote-Repository in deinem Account - sie existiert bisher noch nicht lokal auf deinem Computer. Forks werden verwendet, um eigene Versionen eines Projektes zu erstellen und dann von dort Veränderungen zum originalen Projekt beizutragen (z.B. Bugfixes oder neue Funktionen).

Um ein abgespaltetes Projekt aus deinem GitHub Account auf deinen Computer zu bekommen, wird es geklont. Dieses Klonen kopiert das gesamte Remote-Repository auf deinen Computer, damit du dann lokal daran arbeiten kannst.

Spalte das Patchwork Repository ab

Das Projekt mit dem wir arbeiten werden ist github.com/jlord/patchwork. Öffne die Seite und klicke auf den 'Fork'-Button in der rechten oberen Ecke. Sobald die Animation fertig ist, hast du eine Kopie des Repository in deinem Account gespeichert. Kopiere nun die URL aus der Adresszeile deines Browsers. Das ist die Adresse deines Forks auf dem GitHub-Server.

Klone das abgespaltete Repository auf deinen Computer

Im Terminal wirst du das Repository jetzt auf deinen Computer klonen. Git erzeugt dabei automatisch einen neuen Ordner für das Repository, wir müssen also kein Verzeichnis extra erstellen. Stelle aber sicher, dass du nicht in das Verzeichnis eines anderen Git-Repository hinein klonst!

Versichere dich also, dass du dich nicht innerhalb deines 'hello-world' Verzeichnisses aus den vorherigen Aufgaben befindest. Um das 'hello-world' Verzeichnis zu verlassen und das abgespaltete Repository zu klonen, führe die folgenden Schritte aus:

Verlasse deinen 'hello-world' Ordner:
Tipp: Die beiden Punkte bezeichnen das Verlassen des aktuellen Verzeichnisses um eine Ebene.

cd ..

Da du dich jetzt nicht mehr innerhalb eines anderen Git-Repository befindest, klone dein Repository:

git clone <URLVONGITHUB>

Öffne den Ordner, der für deine lokale Kopie des Repository erzeugt wurde (in diesem Fall der Ordner 'patchwork').

cd patchwork

Du hast jetzt eine lokale Kopie des Repository auf deinem Copmuter, welches automatisch mit dem Remote-Repository in deinem GitHub-Account (die abgespaltete Kopie) verbunden ist. Gib git remote -v ein, um zu sehen, dass die Adresse zum Remote-Repository bereits gespeichert ist.

Verbinde dein Repository mit dem originalen Repository

Was passiert, wenn sich nun das originale Repository verändert? Du wirst dir die Möglichkeit wünschen, diese Änderungen auch in dein abgespaltetes Repository ziehen zu können. Hierfür fügen wir nun noch eine weitere Remote-Verbindung hinzu, diesmal zum originalen github.com/jlord/patchwork Repository mit der passenden URL.

Du kannst diese Remote-Verbindung theoretisch frei benennen, üblicherweiße verwendet man hierfür aber den Begriff 'upstream' (dt. stromaufwärts). Wir möchten diesen Begriff auch hier verwenden.

git remote add upstream https://github.com/jlord/patchwork.git

Um sicherzustellen, dass die Remotes richtig eingestellt sind, gib das Kommando git remote -v ein, um eine Liste der gespeicherten Adressen angezeigt zu bekommen. Du solltest einen 'origin'-Remote mit der Adresse deines Fork, sowie einen 'upstream'-Remote mit der oben genannten, originalen Adresse besitzen.
