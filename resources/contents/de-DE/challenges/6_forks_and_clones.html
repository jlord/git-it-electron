Spalte ein Projekt auf GitHub ab und klone es auf Deinen Computer.

Forks - Abspaltungen

Bisher hast Du ein lokales Projekt neu erstellt und es auf GitHub geschoben. Das ist aber nur der halbe Spaß! Genauso schön ist es mit Anderen Menschen an Projekten zusammenzuarbeiten!

Wenn zu ein Repository abspaltest (engl. fork), erzeugst Du eine Kopie davon in Deinem GitHub Account. Die abgespaltete Kopie beginnt ihre Existenz als ein Remote-Repository in deinem GitHub Account - sie existiert noch nicht auf Deinem Computer. Forks werden verwendet, um eigene Versionen eines Projektes zu erstellen und von dort Veränderungen (wie z.B. Bugfixes oder neue Funktionen) zum originalen Projekt beizutragen.

Um ein abgespaltetes Projekt aus Deinem GitHub Account auf Deinen Computer zu bekommen, wird es geklont. Dieses Klonen kopiert das gesamte Remote-Repository auf Deinen Computer, damit Du dann lokal daran arbeiten kannst.

Spalte das Patchwork Repository ab

Das Projekt mit dem wir arbeiten werden ist github.com/jlord/patchwork. Öffne diese Seite und klicke auf den 'Fork'-Button in der rechten oberen Ecke. Sobald die Animation fertig ist, hast Du eine Kopie des Repository in Deinem Account. Kopiere nun die URL aus der Adresszeile deines Browsers. Das ist die Adresse Deines Forks auf dem GitHub-Server.

Klone Dein Fork auf Deinen Computer

Wieder im Terminal, wirst Du das Repository auf Deinen Computer klonen. Git wird dabei automatisch einen neuen Ordner für das Repository erzeugen, wir müssen also kein Verzeichnis extra erstellen. Stelle aber sicher, dass Du nicht in das Verzeichnis eines anderen Git-Repository hinein klonst!

Versichere Dich also, dass Du Dich nicht innerhalb Deines 'hello-world' Verzeichnisses aus den vorherigen Aufgaben befindest. Um das 'hello-world' Verzeichnis zu verlassen und das abgespaltete Repository zu klonen, folge diesen Schritten:

Verlasse Deinen 'hello-world' Ordner:
Tipp: Die beiden Punkte stehen für das Verlassen des Verzeichnisses um eine Ebene.

cd ..

Da Du Dich jetzt nicht mehr innerhalb eines anderen Git-Repository befindest, klone Dein Fork:

git clone <URLVONGITHUB>

Öffne den Ordner, der für Deine lokale Kopie des Repository erzeugt wurde (in diesem Fall der Ordner 'patchwork').

cd patchwork

Jetzt hast Du eine Kopie des Repository auf Deinem Copmuter, welches automatisch mit dem Remote-Repository in Deinem GitHub-Account (die abgespaltete Kopie) verbunden ist. Gib git remote -v ein, um zu sehen, dass die Adresse zum Remote-Repository bereits gespeichert ist.

Verbinde Dein Repository mit dem Originalen Repository

Was passiert, wenn sich das originale Repository verändert? Du wirst Dir die Möglichkeit wünschen, diese Änderungen auch in dein Repository zu ziehen. Hierfür fügen wir nun noch eine weitere Remote-Verbindung hinzu, diesmal zum originalen github.com/jlord/patchwork Repository mit der passenden URL.

Du kannst diese Remote-Verbindung theoretisch frei benennen, üblicherweiße verwendet man hierfür aber den Begriff 'upstream' (dt. stromaufwärts). Wir möchten diesen Begriff auch hier verwenden.

git remote add upstream https://github.com/jlord/patchwork.git

Um sicherzustellen, dass die Remotes richtig eingestellt sind, gib das Kommando git remote -v ein, um eine Liste der gespeicherten Adressen angzeigt zu bekommen. Du solltest einen 'origin'-Remote mit der Adresse Deines Fork, sowie einen 'upstream'-Remote mit der oben genannten, originalen Adresse besitzen.
