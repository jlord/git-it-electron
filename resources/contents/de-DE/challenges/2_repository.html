Repositories - Verzeichnisse

Ein Repository (dt. Ablage) ist eine Sammlung zusammenhängender Elemente, im Falle der Softwareentwicklung, eine Sammlung von Dateien zum Softwareprojekt. Man kann es sich als einen Projektordner mit allen relevanten Dateien vorstellen. Für den Nutzer wird es letztlich auch als ein solcher Projektordner in Erscheinung treten. Abgekürzt werden "Repositories" auch als "Repos" bezeichnet.

Wenn du nun Git Deinen Projektordner übergibst, wird Git anfangen alle Änderungen in diesem Ordner nachzuverfolgen. Somit wird Dein Projektordner zu einem Git-Repository: Ein Verzeichnis von Dateien, die von Git nachverfolgt werden. Git registriert hierbei wenn Dateien hinzugefügt oder Dateien gelöscht werden und insbesondere auch wenn einzelne Zeichen innerhalb einer Datei verändert werden. All diese Änderungen, inklusive der Informationen Wer diese Änderungen Wann gemacht hat, verfolgt und protokolliert Git. In der Softwareentwicklung wird dieses Verhalten als Versionsverwaltung bezeichnet.

Das Terminal

Das Terminal (auch Bash) ist ein Weg dem Computer Anweisungen mittels schriftlicher Kommandos zu erteilen (statt z.B. durch Mausklick). Im Terminal ist es möglich Dateien zu öffnen, umzubenennen, zu verschieben, Ordner anzulegen, und vieles mehr durch die Eingabe von schriftlichen Kommandos. Das Terminal ist an vielen Stellen sogar mächtiger, als Anwendungen auf grafischer Ebene. Selbst die Textbearbeitung von Code ist mittels Texteditoren (z.B. Vim) direkt im Terminal möglich.

Neben der Verzeichnis-Navigation ist es im Terminal auch möglich Programme zu verwenden, die eine Kommandozeilen-Schnittstelle besitzen ('command-line interface' (CLI)), die also mittels Kommandos im Terminal gesteuert werden können. Git ist ein solches Programm. Der erste Teil des Kommandos sagt dem Computer dabei, dass das Kommando an Git gerichtet ist. Die folgenden Teile sind dann die Kommandos und verschiedenen Optionen, die Git ausführen soll

In Git-it wirst Du ein paar grundlegende Terminal-Kommandos kennen lernen (zusätzlich zu den Git-Kommandos), die für die grundlegende Navigation im Terminal nötig sind. Diese Kommandos sind innerhalb der Schritte beschrieben.

Erzeuge ein Repository

In dieser Aufgabe wirst Du eine neues Projektverzeichnis anlegen und dieses dann als Git-Repository initialisieren. In der Aufgabenstellung nennen wir diesen Ordner 'Hello-World', Du kannst aber auch einen beliebigen anderen Namen verwenden ('Hello-World' ist ein sehr typischer Name für erste Test-Projekte).

Öffne jetzt Dein Terminal!

Gib in deinem Terminal-Fenster die folgenden Befehle nacheinander ein und bestätige diese jeweils mit der Enter-/Eingabe-Taste.

Erstelle zuerst einen neuen Ordner:
Tipp: mkdir ist eine Abkürzung für 'make directory'(dt. Verzeichnis erstellen)

mkdir hello-world

Wechsle Dein Arbeitsverzeichnis in diesen Ordner:
Tipp: cd ist eine Abkürzung für 'change directory'(dt. Verzeichnis wechseln)

cd hello-world

Weiße Git an, das aktuelle Verzeichnis zu initialisieren und Änderungen ab nun nachzuverfolgen:

git init

Das letzte Kommando sollte Dir so etwas wie "Initialized empty Git repository" (dt. Leeres Git-Repository initialisiert) zurückgeben. Die vorherigen Kommandos erzeugen dagegen keine direkte Ausgabe/Rückgabe.

Das war es! Wenn Du möchtest, kannst Du Deine Arbeit nochmals kontrollieren, indem Du git status in die Kommandozeile eingibst. Solange Du keinen Fehler 'fatal: Not a git repository...' (dt. Dies ist kein Git-Repository) zurückbekommst, hast due es geschafft!
