Halte Deine Dateien aktuell, durch ziehen der Änderungen Mitwirkender.

Von einem Remote ziehen

Wenn du mit Anderen gemeinsam an etwas arbeitetes, solltest du mit den letzten Änderungen aktuell bleiben. Änderungen, die auf dem zentralen GitHub-Repository gemacht wurden solltest Du dir also in Dein lokales Repository ziehen (engl. pull).

Was hat Reporobot vor?

Sieh nach, ob Reporobot in Deinem Branch etwas geändert hat, indem Du die Änderungen vom Remote 'origin' auf GitHub ziehst:

git pull <REMOTENAME> <BRANCHNAME>

Falls sich nichts geändert hat, wird Git dir 'Already up-to-date' (dt. bereits aktuell) zurückgeben. Wenn etwas geändert wurde, wird Git diese Änderungen in Deine lokale Version einpflegen.

Hat Reporobot etwas geändert? Git sagt Dir, wo Änderungen erfolgt sind. Du kannst die Datei öffnen und Reporobot's Änderungen anschauen. Überraschung, Reporobot ist ein Künstler!
