Aufgelistet sind die Terminal- und Git-Kommandos, die in den Git-it Aufgaben verwendet werden.

Bash/Terminal Grundlagen

  • Erstelle einen neuen Ordner (aka make directory)
    • mkdir <ORDNERNAME>
  • Navigiere in einen existierenden Ordner (aka change directory)
    • cd <ORDNERNAME>
  • Liste die Elemente im aktuellen Ordner auf
    • ls

Git konfigurieren

  • Überprüfe die Git-Version
    • git --version
  • Speichere deinen Namen
    • git config --global user.name "Dein Name"
  • Speichere deine Email-Adresse
    • git config --global user.email deineemail@beispiel.de
  • Speichere deinen GitHub Account (Groß- & Kleinschreibung beachten!)
    • git config --global user.username NuTzeRnAmE

Git Grundlagen

  • Initialisiere Git für einen Ordner
    • git init
  • Überprüfe den Status deines Repository
    • git status
  • Lass dir die Unterschiede anzeigen
    • git diff
  • Füge die Änderungen an einer Datei zum nächsten Commit hinzu.
    • git add <DATEINAME>
  • Um alle geänderten Dateien hinzuzufügen
    • git add .
  • Übergib die hinzugefügten Änderungen an die Git-(Repository-)Historie, inklusive einer kurzen Nachricht zur Beschreibung der Veränderungen.
    • git commit -m "Deine Nachricht"
  • Klone ein Repository auf deinen Computer
    • git clone <URL>

Git Branches

  • Erzeuge einen neuen Branch
    • git branch <BRANCHNAME>
  • Wähle einen Branch aus
    • git checkout <BRANCHNAME>
  • Du kannst einen Branch mit einem einzigen Kommando erzeugen und auswählen
    • git checkout -b <BRANCHNAME>
  • Liste die verfügbaren Branches auf
    • git branch
  • Benenne den aktuell ausgewählten Branch um
    • git branch -m <NEUERBRANCHNAME>

Git Remotes

  • Füge eine Remote-Verbindung hinzu
    • git remote add <REMOTENAME> <URL>
  • Speichere die URL zu einem existierenden Remote
    • git remote set-url <REMOTENAME> <URL>
  • Zeige die gespeicherten Remote-Adressen an
    • git remote -v

Ziehen(pull) vom Git-Server

  • Änderungen ziehen
    • git pull
  • Ziehe Änderungen von einem Remote Branch
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • Schaue dir die Änderungen an, bevor du sie vom Remote ziehst
    • git fetch --dry-run

Schieben(push) & Verbinden(merge) in Git

  • Schiebe Änderungen auf den Server
    • git push <REMOTENAME> <BRANCHNAME>
  • Integriere einen Branch in den aktuellen Branch
    • git merge <BRANCHNAME>

Remotes und Branches löschen

  • Lösche den lokalen Branch
    • git branch -d <BRANCHNAME>
  • Lösche einen Remote-Branch
    • git push <REMOTENAME> --delete <BRANCHNAME>
