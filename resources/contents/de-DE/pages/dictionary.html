Aufgelistet sind die Terminal und Git-Kommandos, die in den Git-it Aufgaben verwendet werden.

Bash/Terminal Grundlagen

  • Erzeuge einen neuen Ordner (aka make directory)
    • mkdir <ORDNERNAME>
  • Navigiere in einen vorhandenen Ordner (aka change directory)
    • cd <ORDNERNAME>
  • Erstelle eine Liste der Elemente in einem Ordner
    • ls

Git konfigurieren

  • Überprüfe die Git-Version
    • git --version
  • Speichere Deinen Namen
    • git config --global user.name "Dein Name"
  • Speichere Deine Email-Adresse
    • git config --global user.email deineemail@beispiel.de
  • Speichere Deinen GitHub Account (Groß- & Kleinschreibung beachten!)
    • git config --global user.username NuTzeRnAmE

Git Grundlagen

  • Aktiviere Git für einen Ordner
    • git init
  • Überprüfe den Status der Veränderungen in einem Repository
    • git status
  • Zeige die Veränderungen
    • git diff
  • Füge die Veränderungen einer Datei zum nächsten Commit hinzu
    • git add <DATEINAME>
  • Um alle Änderungen hinzuzufügen
    • git add .
  • Um die hinzugefügten Änderungen zu übergeben (zu speichern) - inklusive einer kurzen Nachricht, die die Änderungen beschreibt
    • git commit -m "Deine Nachricht"
  • Klone ein Repository auf Deinen Computer
    • git clone <URL>

Git Branches

  • Erzeuge einen neuen Branch
    • git branch <BRANCHNAME>
  • Wähle einen Branch aus
    • git checkout <BRANCHNAME>
  • Du kannst einen Branch mit einem Befehl erstellen und direkt auswählen
    • git checkout -b <BRANCHNAME>
  • Liste die Branches auf
    • git branch
  • Benenne den aktuellen Branch um
    • git branch -m <NEUERBRANCHNAME>

Git Remotes

  • Füge eine Remote-Verbindung hinzu
    • git remote add <REMOTENAME> <URL>
  • Speichere die URL zu einem Remote
    • git remote set-url <REMOTENAME> <URL>
  • Zeige die Remote-Verbindungen
    • git remote -v

Ziehen(pull) vom Git-Server

  • Änderungen ziehen
    • git pull
  • Änderungen von einem Remote-Branch ziehen
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • Die Veränderungen des Remote anzeigen (vor dem ziehen)
    • git fetch --dry-run

Schieben(push) & Verbinden(merge) in Git

  • Änderungen zum Remote schieben
    • git push <REMOTENAME> <BRANCHNAME>
  • Einen Branch mit dem aktuellen Branch verbinden
    • git merge <BRANCHNAME>

Remotes und Branches löschen

  • Lösche einen lokalen Branch
    • git branch -D <BRANCHNAME>
  • Lösche einen Remote-Branch
    • git push <REMOTENAME> --delete <BRANCHNAME>
