Alle verwendeten Terminal und Git-Befehle aus den Git-it Herausforderungen.

Bash/Terminal Grundlagen

  • Neuer Ordner erstellen (aka make directory)
    • mkdir <FOLDERNAME>
  • In existenten Ordner navigieren / springen (aka change directory)
    • cd <FOLDERNAME>
  • Alle Elemente im Ordner auflisten
    • ls

Git Konfigurieren

  • Git Version ausgeben
    • git --version
  • Name für Commits setzen
    • git config --global user.name "Your Name"
  • E-Mail-Adresse für Commits setzen
    • git config --global user.email youremail@example.com
  • GitHub Benutzername setzen (Groß-/Kleinschreibung beachten)
    • git config --global user.username uSeRnAmE

Git Grundlagen

  • Git in einem Ordner initialisieren
    • git init
  • Änderungen für ein Repository ansehen
    • git status
  • Änderungen an einer Datei ansehen
    • git diff
  • Änderungen an einer Datei zu Commit hinzufügen
    • git add <FILENAME>
  • Alle Änderungen zu Commit hinzufügen
    • git add .
  • Commiten (aka speichern) der Änderungen mit einer Nachricht
    • git commit -m "your commit message"
  • Repository auf den Computer kopieren
    • git clone <URL>

Git Branches

  • Neuer Branch erstellen
    • git branch <BRANCHNAME>
  • Zu Branch wechseln
    • git checkout <BRANCHNAME>
  • Neuer Branch erstellen & direktes wechseln
    • git checkout -b <BRANCHNAME>
  • Alle Branches auflisten
    • git branch
  • Aktueller Branch umbenennen
    • git branch -m <NEWBRANCHNAME>

Git Remotes

  • Neue Remote Verbindung (aka GitHub) hinzufügen
    • git remote add <REMOTENAME> <URL>
  • Die URL der Remote Verbindung (aka GitHub) setzen
    • git remote set-url <REMOTENAME> <URL>
  • Remote Verbindungen auflisten
    • git remote -v

Pulling in Git

  • Pull in changes
    • git pull
  • Pull in changes from a remote branch
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • See changes to the remote before you pull in
    • git fetch --dry-run

Push & Merge in Git

  • Änderungen auf Remote (aka GitHub) veröffentlichen
    • git push <REMOTENAME> <BRANCHNAME>
  • Branch mit aktuellem Branch zusammenführen
    • git merge <BRANCHNAME>

Remote Branches löschen

  • Lokaler Branch löschen
    • git branch -D <BRANCHNAME>
  • Remote Branch löschen (aka Branch auf GitHub)
    • git push <REMOTENAME> --delete <BRANCHNAME>
