Poniżej znajduje się spis poleceń terminala i Gita używanych w wyzwaniach Git-it.

Podstawy Bash/Terminala

  • Tworzenie nowego folderu (aka tworzenie nowego katalogu)
    • mkdir <NAZWAKATALOGU>
  • Nawigacja do istniejącego folderu (aka zmiana katalogu)
    • cd <NAZWAKATALOGU>
  • Wypisanie zawartości folderu
    • ls

Konfiguracja Git

  • Sprawdzenie wersji Git
    • git --version
  • Ustawienie swojej nazwy
    • git config --global user.name "Twoja nazwa"
  • Ustawienie swojego adresu e-mail
    • git config --global user.email twójemail@example.com
  • Ustawienie swojej nazwy GitHub (wielkość znaków ma znaczenie)
    • git config --global user.username uSeRnAmE

Podstawy Git

  • Włączanie Git dla bieżącego katalogu
    • git init
  • Sprawdzenie statusu zmian w repozytorium
    • git status
  • Wyświetlenie zmian w plikach
    • git diff
  • Dodawanie zmian do obszaru zatwierdzania
    • git add <NAZWAPLIKU>
  • Dodawanie wszystkich zmian do obszaru zatwierdzania
    • git add .
  • Zatwierdzenie (zapisanie) zmian wybranych do zatwierdzenia wraz z krótką informacją je opisującą
    • git commit -m "krótki opis zmian"
  • Skopiowanie (sklonowanie) repozytorium na Twój komputer
    • git clone <URL>

Gałęzie

  • Tworzenie nowej gałęzi
    • git branch <NAZWAGAŁĘZI>
  • Przełączanie aktywnej gałęzi
    • git checkout <NAZWAGAŁĘZI>
  • Tworzenie nowej gałęzi i przełączenie na nią jednym poleceniem
    • git checkout -b <NAZWAGAŁĘZI>
  • Wypisanie listy gałęzi
    • git branch
  • Zmiana nazwy aktywnej gałęzi
    • git branch -m <NOWANAZWAGAŁĘZI>

Repozytoria zdalne

  • Dodawanie repozytorium zdalnego
    • git remote add <NAZWAREPOZDALNEGO> <URL>
  • Ustawienie URL do repozytorium zdalnego
    • git remote set-url <NAZWAREPOZDALNEGO> <URL>
  • Przeglądanie listy repozytoriów zdalnych
    • git remote -v

Pobieranie zmian

  • Pobieranie zmian
    • git pull
  • Pobieranie zmian z gałęzi w repozytorium zdalnym
    • git pull <NAZWAREPOZDALNEGO> <GAŁĄŹZDALNA>
  • Wyświetlenie zmian z repozytorium zdalnego przed ich pobraniem
    • git fetch --dry-run

Wypychanie i scalanie

  • Wypychanie zmian
    • git push <NAZWAREPOZDALNEGO> <NAZWAGAŁĘZI>
  • Scalanie gałęzi o podanej nazwie do aktywnej gałęzi
    • git merge <NAZWAGAŁĘZI>

Usuwanie gałęzi lokalnych i zdalnych

  • Usuwanie gałęzi lokalnej
    • git branch -D <NAZWAGAŁĘZI>
  • Usuwanie gałęzi zdalnej
    • git push <NAZWAREPOZDALNEGO> --delete <NAZWAGAŁĘZI>
