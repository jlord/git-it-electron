Scalanie gałęzi lokalnie, usunięcie gałęzi i pobieranie z repozytorium oryginalnego.

Sprzątanie

Kiedy gałęzie nie są Ci już potrzebne możesz usunąć je z repozytorium lokalnego i zdalnego. Jest to pomocne, gdyż repozytorium nie zostaje zaśmiecone starymi gałęziami.

Scalanie lokalnie

Twój pull request został scalony! Twoja gałąź została scalona do gałęzi 'gh-pages' w oryginalnym repozytorium na GitHubie. Możesz również scalić swoją gałąź lokalnie.

Najpierw, przejdź na gałąź do której chcesz scalić zmiany z innej gałęzi — w tym wypadku będzie to gałąź 'gh-pages'.

git checkout gh-pages

Teraz poinstruuj Gita która gałąź będzie źródłem scalanych zmian — w tym wypadku będzie to gałąź, której nazwa zaczyna się od "add-".

git merge <NAZWAGAŁĘZI>

Posprzątaj po sobie usuwając scaloną już gałąź. Gdy została scalona nie będzie już więcej potrzebna.

git branch -d <NAZWAGAŁĘZI>

Możesz również usunąć gałąź z Twojego repozytorium zdalnego na GitHubie:

git push <NAZWAREPOZYTORIUM> --delete <NAZWAGAŁĘZI>

Pobieranie z repozytorium oryginalnego

Ostatnią, ale nie mniej ważną informacją jest fakt, że oryginalne repozytorium zmieniło się ponieważ Twój pull request został zaakceptowany — Reporobot dodał Twoją nazwę do strony! Jeśli pobierzesz te zmiany z repozytorium oryginalnego będziesz posiadał aktualną kopię repozytorium. Możesz również zobaczyć ją na żywo pod adresem: twojanazwaużytkownika.github.io/patchwork.

Aby pobrać zmiany z oryginalnego repozytorium:

git pull upstream gh-pages
