Tworzenie pliku w Twoim nowym repozytorium, dodawanie mu zawartości i zatwierdzenie zmian za pomocą Git.

Zatwierdzenia zmian

Zatwierdzenia zmian (potocznie zwane commitami od angielskiego Commits) są kluczem do korzystania z Gita. Są to momenty w których zapisywałeś i opisywałeś wykonaną pracę. To punkty na osi czasu Twojego projektu.

Tworzenie nowego pliku

Teraz gdy masz już skonfigurowane repozytorium dodaj do niego plik.

Otwórz swój edytor tekstowy i utwórz nowy pusty plik. Teraz wprowadź do niego zawartość, może podstawowe przywitanie "Cześć!", i zapisz go jako 'readme.txt' w folderze 'hello-world' utworzonym w ostatnim wyzwaniu.

Sprawdzanie statusu + dodawanie i zatwierdzanie zmian

Następnie sprawdź status repozytorium by dowiedzieć się czy miały miejsce zmiany. Ty wiesz o tym kiedy coś zmieniasz, ale czy Git wie?

Upewnij się, że nadal jesteś w folderze 'hello-world' kiedy będziesz uruchamiał poniższe polecenia. Za pomocą Gita sprawdź co się zmieniło w repozytorium:

Najpierw sprawdź status repozytorium:

git status

Git powinien powiedzieć Ci, że plik został dodany.

Następnie dodaj dopiero utworzony plik do listy zmian do zatwierdzenia (zapisania) za pomocą Gita:

git add readme.txt

Na koniec, zatwierdź wybrane zmiany w historii repozytorium wraz z krótką wiadomością opisującą zawartość aktualizacji.

git commit -m "Utworzono readme"

Krok: zrób więcej zmian

Dodaj kolejną linię do 'readme.txt' i po raz kolejny zapisz plik.

W terminalu możesz zobaczyć różnicę pomiędzy plikiem teraz, a ostatnim zatwierdzonym stanem.

Poinstruuj gita by wyświetlił różnice:

git diff

Wykorzystując wiedzę z poprzedniego kroku, zapisz bieżące zmiany.
