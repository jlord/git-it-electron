Pull Requesty

Często kiedy wprowadzasz zmiany i usprawnienia do projektu który sforkowałeś, będziesz chciał wysłać te zmiany właścicielowi oryginalnego projektu i poprosić o ich scalenie by wszyscy mogli z nich skorzystać. Tym właśnie jest pull request.

Chcielibyśmy dodać Cię do listy wykańczających warsztat, więc załóż pull request w celu dodania pliku do oryginalnego repozytorium: github.com/jlord/patchwork.

Tworzenie pull requesta

Przejdź na stronę oryginału sforkowanego przez Ciebie repozytorium na GitHubie, w tym przypadku http://github.com/jlord/patchwork.

Często GitHub wykryje wypchnięcie gałęzi do forka i wyświetli ją na szczycie oryginalnego repozytorium. Jeśli widzisz swoją gałąź 'add-nazwaużytkownika', możesz kliknąć by utworzyć pull request z poziomu tego widoku. Jeśli nie:

  • Kliknij zielony przycisk 'New pull request'.
  • Wybierz gałąź zawierającą zmiany, które chcesz przesłać. Powinna być to ta nazwana 'add-nazwaużytkownika'.

Twoim oczom ukaże się strona ze szczegółami pull requesta, który jest właśnie tworzony. Ta strona pokazuje zatwierdzenia i zmiany w formie różnic, związane z Twoim pull requestem w porównaniu do gałęzi 'gh-pages' w oryginalnym repozytorium.

Jeśli oryginalne repozytorium posiada dokumentację kontrybucji, GitHub odwoła się do niej. To dokumentacja pochodząca od właścicieli repozytorium objaśniająca jak pracować na rzecz projektu by osiągnąć najlepszy efekt — bardzo przydatna jeśli chcesz by Twoje zmiany zostały zaadoptowane!

Jeśli cała zawartość strony wygląda dobrze i tak jak oczekujesz:

  • Kliknij 'Create pull request'
  • Dodaj tytuł i opis zmian, których scalenie sugerujesz.
  • Kliknij 'Send pull request'!

Bingo! Wysłałeś pull requesta — weź głęboki oddech i pław się w chwale.

Jeśli wszystko przebiegło pomyślnie i Twój pull request zostanie uznany za wartościowy, będzie scalony w kilka chwil. Jeśli nie zostanie scalony w ciągu kilku minut, prawdopodobnie otrzymasz informację zwrotną od @reporobot na temat przyczyn odrzucenia tych zmian. Jeśli tak się stało, zamknij utworzony pull request na GitHub, wprowadź wymagane zmiany do swojej gałęzi, wypchnij te zmiany z powrotem do swojego forka i otwórz ponownie pull request (to sprawi, że @reporobot zweryfikuje zmiany ponownie).
