Dodawanie gałęzi lokalnie do sforkowanego repozytorium by pracować nad własnymi zmianami.

Gałęzie

Repozytoria Git korzystają z gałęzi do odizolowania pracy, gdy jest to wymagane. Tworzenie gałęzi jest powszechną praktyką gdy współpracujesz z innymi osobami nad projektem. W ten sposób możesz kontynuować pracę podczas gdy gałąź główna (często nazywana 'master') posiada stabilną wersję. Kiedy ukończysz pracę, scalasz ją do głównej gałęzi.

Diagram poniżej pokazuje 'rozgałęzianie' od głównej gałęzi 'master', wykonanie pracy, a następnie scalenie zmian z powrotem do 'master'. Nie ma wymagania, że źródłem dla nowej gałęzi musi być 'master' - może to być dowolna inna gałąź.

Dla lepszej wizualizacji sposobu działania gałęzi w projektach zapoznaj się z tym poradnikiem GitHuba: guides.github.com/overviews/flow

GitHub Pages

GitHub automatycznie prezentuje i hostuje strony internetowe w gałęziach nazwanych 'gh-pages'. Ta darmowa usługa nosi nazwę GitHub Pages. Ponieważ projekt którego forka utworzyłeś tworzy stronę, jego główna gałąź nazwana została 'gh-pages' zamiast 'master'. Wszystkie repozytoria zawierające gałąź 'gh-pages' ze stroną mogą zostać odnalezione w sieci przy użyciu poniższego wzorca adresu URL:

http://nazwaużytkownika.github.io/nazwarepozytorium

Tworzenie gałęzi

Kiedy tworzysz gałąź git kopiuje całą zawartość bieżącej gałęzi (tej, na której jesteś) i umieszcza ją na nowej gałęzi.

Będąc wciąż w lokalnym repozytorium 'patchwork' repository, wpisz git status by sprawdzić na jakiej gałęzi jesteś obecnie. Git powinien odpowiedzieć, że jest to gałąź 'gh-pages'.

Teraz utwórz nową gałąź i nadaj jej nazwę "add-<nazwaużytkownika>", gdzie 'nazwaużytkownika' to Twoja nazwa użytkownika w serwisie GitHub, na przykład "add-jlord". Ponieważ nazwa gałęzi rozróżnia duże i małe litery pamiętaj o wprowadzeniu nazwy użytkownika dokładnie tak, jak widnieje w GitHub.

git branch <NAZWAGAŁĘZI>

Posiadasz nową gałąź oraz starą o nazwie 'gh-pages'.

Aby przełączyć się na nowo utworzoną gałąź i na niej pracować należy użyć opcji checkout. Przełącz się na nową gałąź:

git checkout <NAZWAGAŁĘZI>

Krok: Tworzenie nowego pliku

Wróć do swojego ulubionego edytora tekstowego:

  • Utwórz nowy plik nazwany "add-<NAZWAUŻYTKOWNIKA>.txt", gdzie 'NAZWAUŻYTKOWNIKA' jest Twoją nazwą użytkownika, na przykład "add-jlord.txt".
  • Następnie, jako jego zawartość wpisz swoją nazwę użytkownika w GitHub, tylko tyle i aż tyle. Przykładowo, wpisałbym 'jlord'.
  • Zapisz ten plik w folderze 'contributors' w repozytorium lokalnym Patchwork: Patchwork/contributors/add-NAZWAUŻYTKOWNIKA.txt
  • Następnie zapisz zmiany w repozytorium (zobacz poniżej).

Zapisywanie zmian

Przejdź przez poniższe kroki by zapisać swoje zmiany:

git status

git add <contributors/NAZWAPLIKU>

git commit -m "informacja opisująca zmiany"

Teraz wypchnij swoje zmiany do swojego forka, 'origin', na GitHubie:

git push origin <NAZWAGAŁĘZI>

Permission denied...error: 403

Wypychasz zmiany do repozytorium, w którym nie masz prawa zapisu. W tym wypadku, prawdopodobnie próbujesz wypchnąć zmiany do oryginalnego repozytorium 'jlord/patchwork'. Upewnij się, że 'pushujesz' do 'origin' oraz że repozytorium zdalne o tej nazwie kieruje do Twojego forka w GitHub. By sprawdzić listę repozytoriów zdalnych wraz z ich adresami uruchom polecenie git remote -v w terminalu. Powinieneś mieć repozytorium zdalne 'upstream' kierujące do 'jlord/patchwork' oraz 'origin' kierujące do 'NAZWAUŻYTKOWNIKA/patchwork'.

By naprawić połączenia z repozytoriami zdalnymi kierujące pod niewłaściwy adres możesz ustawić ich adres URL ponownie: git remote set-url origin .

Plik NIE znajdujący się w folderze 'contributors'

Utworzony przez Ciebie plik powinien być umieszczony w istniejącym folderze 'contributors' w repozytorium Patchwork. Jeśli umieściłeś go gdzieś indziej, użyj Findera albo Eksploratora Windows by przenieść go we właściwe miejsce. Możesz zweryfikować status repozytorium ponownie za pomocą git status by zobaczyć swoje zmiany. Dodaj wszystkie (-A) zmiany do zatwierdzenia, a następnie zatwierdź je używając poleceń poniżej.

git add -A

git commit -m "przeniesienie pliku do folderu contributors"

Branch name expected: _____

Nazwa gałęzi powinna dokładnie odpowiadać Twojej nazwy użytkownika w GitHub. Zmień nazwę gałęzi:

git branch -m <NOWANAZWAGAŁĘZI>

Kiedy poczynisz zmiany zweryfikuj je ponownie!
