Ниже приведены команды терминала и Git, используемые в задачах Git-it.

Основы Bash/Терминала

  • Создание новой папки (также известное как создание нового каталога)
    • mkdir <FOLDERNAME>
  • Переход в существующую папку (также известное как изменение каталога)
    • cd <FOLDERNAME>
  • Показать список элементов в папке
    • ls

Настройка Git

  • Проверить версию Git
    • git --version
  • Укажите свое имя
    • git config --global user.name "Your Name"
  • Укажите свой адрес электронной почты
    • git config --global user.email youremail@example.com
  • Установите свою учетную запись Github (с учетом регистра)
    • git config --global user.username uSeRnAmE

Основы Git

  • Включите Git для папки
    • git init
  • Проверить статус изменений в репозитории
    • git status
  • Просмотр изменений в файлах
    • git diff
  • Добавить изменения файла для фиксации/коммита
    • git add <FILENAME>
  • Добавить все изменения в файлах
    • git add .
  • Для фиксации (или сохранения) добавленных вами изменений с помощью короткого сообщения с описанием изменений
    • git commit -m "your commit message"
  • Скопируйте репозиторий на свой компьютер
    • git clone <URL>

Ветви Git

  • Создать новую ветку
    • git branch <BRANCHNAME>
  • Перейти на ветку
    • git checkout <BRANCHNAME>
  • Вы можете создать ветку и переключиться на нее одной строкой
    • git checkout -b <BRANCHNAME>
  • Показать список веток
    • git branch
  • Переименуйте ветку, в которой вы сейчас находитесь
    • git branch -m <NEWBRANCHNAME>

Удаленные репозитории Git

  • Добавить удаленные подключения
    • git remote add <REMOTENAME> <URL>
  • Установить URL-адрес на удаленный репозиторий
    • git remote set-url <REMOTENAME> <URL>
  • Посмотреть список удаленных подключений
    • git remote -v

Получение данных в Git

  • Получение изменений
    • git pull
  • Получение изменений из удаленной ветки
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • Посмотреть изменения в удаленной ветке перед его получением
    • git fetch --dry-run

Отправка & Слияние в Git

  • Отправка изменений
    • git push <REMOTENAME> <BRANCHNAME>
  • Слиять ветку в текущую ветку
    • git merge <BRANCHNAME>

Удаление удаленных репозиторий и веток в Git

  • Удаление локальной ветки
    • git branch -D <BRANCHNAME>
  • Удаление удаленной ветки
    • git push <REMOTENAME> --delete <BRANCHNAME>
