Репозитории

Репозиторий - это набор связанных элементов. В нашем случае при написании программного обеспечения это набор файлов, связанных с проектом программного обеспечения. Вы можете представить это как папку проекта со всеми соответствующими файлами внутри нее. Фактически, в любом случае это будет выглядеть на вашем компьютере именно так. Иногда их для краткости называют "repos".

Вы сообщаете Git, что это за проект, и Git начинает отслеживать все изменения в этой папке. Это делает его репозиторием Git: папкой элементов, отслеживаемых Git. Git отслеживает, когда добавляются файлы, удаляется или даже изменяется одна буква в одном файле. Кроме этого Git отслеживает кто и когда это сделал. В программном обеспечении отслеживание подобных изменений называется управление версиями .

Использование терминала

Терминал (или Bash) - это способ указать компьютеру выполнять действия путем ввода команд, а не щелчка мышью по приложениям. Вы можете переименовывать файлы, открывать файлы, создавать новые папки, перемещаться между каталогами (папками) и многое другое, выполняя вводимые команды. Вы даже можете использовать текстовый редактор для кода (например, Vim ) в своем терминале и никогда не покидать его!

Помимо навигации по компьютеру, вы также можете использовать программы в Терминале, которые имеют интерфейс командной строки (CLI), то есть их можно запускать с помощью команд в терминале. Git - один из них. Первая часть команды сообщает вашему компьютеру, что вы разговариваете с Git. Следующие за ними части - это команды и различные параметры, над которыми должен работать Git.

В Git-it вы изучите несколько основных действий командной строки (в дополнение к командам Git) для навигации по вашему компьютеру; они описаны в следующих шагах.

Создание репозиторий

Вы собираетесь создать новую папку проекта, а затем инициализировать ее как репозиторий Git. В этом задании мы назовем папку «hello-world», но вы можете выбрать другое имя, если у вас уже есть папка с таким именем (это обычное имя первого проекта).

Пора открыть свой терминал!

В окне терминала введите эти команды по одной, нажимая клавишу ввода/возврата после каждой команды.

Сначала создайте новую папку:
Совет: mkdir означает make directory (создать каталог)

mkdir hello-world

Затем перейдите в эту папку:
Совет: cd означает change directory (изменить каталог)

cd hello-world

Наконец, скажите Git инициализировать (начать отслеживание) папку, в которой вы сейчас находитесь:

git init

Последняя команда должна возвращать что-то, наподобие "Initialized empty Git repository" ("Инициализирован пустой репозиторий Git"). Остальные команды ничего не возвращают.

Вы сделали это! Если вы хотите быть дважды уверены, что это репозиторий Git, введите git status , и если он не вернет 'fatal: Not a git repository...', вы молодец!
