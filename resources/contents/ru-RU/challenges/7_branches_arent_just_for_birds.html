Добавьте ветку локально в свой разветвленный репозиторий для работы с вашими изменениями.

Ветви

Репозитории Git используют ветки, чтобы изолировать работу, когда это необходимо. Обычной практикой при работе над проектом или совместно с другими участниками проекта является создание ветки , чтобы сохранить ваши рабочие изменения. Таким образом, вы можете выполнять свою работу, пока основная, обычно называемая 'master', ветвь остается стабильным. Когда работа над вашей веткой закончена, вы снова объединяете ее с веткой 'master'.

На диаграмме ниже показано, как вы можете отделиться от своей ветки 'master', выполнить работу, а затем объединить эти изменения обратно в ветку 'master'. Вы даже можете ответвиться от ветки, если вам нужно, ветвь 'master' не обязательно должна быть базовой.

Чтобы лучше понять, как работают ветки в проекте, см. это руководство GitHub: guides.github.com/overviews/flow

GitHub Pages

GitHub будет автоматически обслуживать и размещать файлы статических веб-сайтов в ветках с именем 'gh-pages'. Эта бесплатная служба называется GitHub Pages . Поскольку разветвленный проект создает веб-сайт, его основная ветвь называется 'gh-pages' вместо 'master'. Все репозитории, в которых есть ветка 'gh-pages' с файлами веб-сайтов, можно найти в интерактивном режиме, используя этот шаблон для URL:

http://githubusername.github.io/repositoryname

Создание ветки

Когда вы создаете ветку, Git копирует все из текущей ветки, в которой вы находитесь, и помещает ее в ветку, которую вы просили создать.

Находясь в локальном репозитории 'patchwork', введите git status , чтобы узнать, в какой ветке вы сейчас находитесь. Git должен ответить, что вы находитесь в ветке 'gh-pages'.

Теперь создайте новую ветку и назовите ее "add-<username>", где username - ваше имя пользователя. Например, "add-jlord". Ветви чувствительны к регистру, поэтому назовите ветку точно так, как отображается ваше имя GitHub .

git branch <BRANCHNAME>

Теперь у вас есть ветка с новым именем, идентичным 'gh-pages'.

Чтобы перейти в эту ветку и поработать над ней, вы должны переключиться на ветку. Зайдите в свою новую ветку:

git checkout <BRANCHNAME>

Шаг: Создайте новый файл

В текстовом редакторе:

  • Создайте новый файл с именем "add-<USERNAME>.txt", где 'USERNAME' - ваше имя пользователя. Например, "add-jlord.txt".
  • Затем просто напишите в нем свое имя пользователя GitHub, вот и все. Например, я бы набрал 'jlord'.
  • Сохраните это в файле 'contributors' в папке Patchwork: Patchwork/contributors/add-yourusername.txt
  • Затем проверьте свои изменения (см. ниже).

Check-in

Выполните шаги для регистрации проекта:

git status

git add <contributors/FILENAME>

git commit -m "commit message"

Теперь отправьте обновление в ваш форк , 'origin', на GitHub:

git push origin <BRANCHNAME>

Отказано в доступе ... ошибка: 403

Вы отправляете изменения в репозиторий, к которому у вас нет доступа на запись. В этом случае вы, скорее всего, отправляете к исходному 'jlord/patchwork'. Убедитесь, что вы отправляете на 'origin' и что он указывает на адрес вашего форма на GitHub. Чтобы проверить и увидеть, какие у вас есть удаленные репозитории и куда они указывают, выполните git remote -v . У вас должно быть 'upstream', указывающее на 'jlord/patchwork', и 'origin', указывающее на 'yourusername/patchwork'.

Чтобы исправить удаленный репозиторий, указывающий не на то место, вы можете повторно установить его URL:git remote set-url origin.

Ошибка аутентификации...ошибка: 401

Не удалось подтвердить вашу личность. У вас может быть включена 2FA (двухфакторная аутентификация), и в этом случае вы должны использовать токен личного доступа в качестве пароля. Вы можете создать его, следуя этим инструкциям . Вам нужно будет сохранить его и использовать вместо пароля при появлении запроса. Вы также можете сохранить этот токен доступа на своем компьютере, чтобы не нужно было вводить его повторно.

Файл НЕ в папке 'contributors'

Созданный вами файл должен быть помещен в существующую папку 'contributors' в репозитории Patchwork. Если вы поместили его в другое место, просто используйте Finder или проводник Windows, чтобы переместить файл в папку. Вы можете снова проверить git status , и вы обнаружите, что он видит ваши изменения. Подготовьте, а затем зафиксируйте «все» (-A) эти изменения (добавлений и удалений) с помощью команд ниже.

git add -A

git commit -m "move file into contributors folder"

Ожидается название ветви: _____

Название ветки должно точно соответствовать вашему имени пользователя. Чтобы изменить название вашей ветки:

git branch -m <NEWBRANCHNAME>

Когда вы сделали ваши обновления, проверьте еще раз!

  • Вы можете создать ветку и переключиться на нее одной строкой
  • git checkout -b <BRANCHNAME>
  • Создать новую ветку
  • git branch <BRANCHNAME>
  • Перейти на ветку
  • git checkout <BRANCHNAME>
  • Перечислить ветки
  • git branch
  • Переименовать ветку, в которой вы сейчас находитесь
  • git branch -m <NEWBRANCHNAME>
  • Проверить ветку, на которой вы работаете
  • git status
