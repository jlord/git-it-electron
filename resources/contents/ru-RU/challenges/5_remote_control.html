Подключите локальный репозиторий к удаленному и отправьте в него изменения.

Удаленные репозитория

Когда вы размещаете что-то на GitHub, эта копия находится на одном из серверов GitHub. Это делает его удаленным репозиторием, потому что он находится не на вашем компьютере, а на сервере, «удаленном» или другом месте. Отправляя на него локальные (на своем компьютере) изменения, вы поддерживаете его актуальность.

Другие пользователи всегда могут получить последнюю информацию о вашем проекте, загрузив ваши изменения с удалленого репозитория (на свой компьютер). Таким образом каждый может работать над проектом вместе, не имея доступа к вашему компьютеру, на котором хранится ваша локальная копия.

Создайте удаленный репозиторий

Вы хотите синхронизировать свою локальную версию с версией, хранящейся на GitHub.com. Сначала создайте новый удаленный репозиторий на GitHub.com.

  • Перейдите на github.com , войдите в систему, нажмите "+" в правом верхнем углу, а затем нажмите "Создать репозиторий".
  • Дайте ему название и краткое описание, которое соответствует имени вашего локального репозитория, 'hello-world'.
  • Сделайте его общедоступным. Это означает, что оно будет доступно в вашем общедоступном профиле.
  • Не инициализируйте файл README, потому что у нас уже есть локальный файл с именем readme.txt. Это вспомогательный вариант от GitHub, если вы еще этого не сделали.
  • Оставьте значения ".gitignore" и "license" "none". Мы не будем использовать их в этом руководстве.
  • Нажмите "Создать репозиторий"!

Readme, .gitignore и лицензии

Это обычные файлы в проектах с открытым исходным кодом, поэтому для облегчения запуска нового проекта GitHub предоставляет возможность их создания.

Файл readme объясняет, что это за проект, как его использовать и часто, как внести свой вклад (хотя иногда для этих деталей есть дополнительный файл CONTRIBUTING.md).

Файл .gitignore - это список файлов, которые Git не должен отслеживать, например файлы с паролями!

Файл лицензии - это тип лицензии, которую вы используете для своего проекта. Это позволяет другим узнать, как они могут его использовать. Информация о типах лицензии находится здесь: choosealicense.com .

Подключите локальный компьютер к удаленному репозиторию

Теперь у вас создан пустой репозиторий на GitHub.com. Вверху вы увидите ссылку «Быстрая установка», убедитесь, что выбрана кнопка «HTTPS», и скопируйте адрес - это адрес вашего репозитория на серверах GitHub.

Вернувшись в свой терминал и внутри папки 'hello-world', которую вы инициализировали как репозиторий Git в предыдущем задании, вы хотите сказать Git, чтобы он запомнил адрес удаленной версии на серверах GitHub. У вас может быть несколько удалленных репозиторий, поэтому для каждого требуется имя. Основной удалленный репозиторий обычно называется origin .

Чтобы добавить удалленный репозиторий с именем 'origin' в ваш репозиторий:

git remote add origin <URLFROMGITHUB>

Вашему локальному репозиторию теперь известно, где находится ваш удаленный репозиторий с именем 'origin' на серверах GitHub. Думайте об этом как о добавлении имени и адреса на быстрый набор - теперь, когда вам нужно что-то отправить туда, вы можете сделать это.

Пользователи Windows:

Если на вашем компьютере установлен GitHub Desktop , в вашем локальном репозитории автоматически создается удаленный репозиторий с именем 'origin'. В этом случае вам просто нужно указать, какой URL-адрес ассоциировать с 'origin'. Используйте эту команду вместо приведенной выше команды "add":

git remote set-url origin <URLFROMGITHUB>

Отправьте работу на удаленный репозиторий

Далее вам необходимо отправить все, что вы сделали локально, в удаленный репозиторий на GitHub. Это то, что вы будете делать часто, чтобы ваша удаленная версия была актуальной и соответствовала состоянию вашей локальной версии.

В Git есть система ветвления, для того чтобы вы могли работать над разными частями проекта в разное время. Мы ознакомимся с ним позже, но по умолчанию первая ветка называется 'master'. Когда вы отправляете (а затем получаете) из проекта, вы сообщаете Git имя ветки , которое вы хотите, и имя удаленной ветки , на котором он находится.

В этом случае мы отправим нашу ветку с именем 'master' на наш удаленный репозиторий с именем 'origin' на GitHub.

git push origin master

Теперь перейдите на страницу удаленного репозитория на GitHub.com и обновите страницу. Ух ты! Теперь все одинаково - как локально, так и удаленно. Поздравляем с получением первого публичного репозитория!
