Коммиты

Коммиты - это основа использования Git. Это моменты, когда вы сохраняете и описываете проделанную работу. Это отметки на шкале времени истории вашего проекта.

Создайте новый файл

Теперь, когда у вас есть репозиторий, давайте добавим в него файл.

Откройте текстовый редактор и создайте новый пустой файл. Теперь напишите немного текста, например, введите «Hello!» И сохраните файл как «readme.txt» в папке «hello-world», которую вы создали в предыдущем задании.

Проверить статус + добавить и зафиксировать изменения

Затем проверьте статус вашего репозитория, чтобы узнать, были ли в нем изменения. Вы знаете, что что-то изменили, но знает-ли Git?

Убедитесь, что вы все еще находитесь в своей директории 'hello-world', когда выполняете эти команды. Используйте Git, чтобы узнать, что изменилось в вашем репозитории:

Сначала проверьте статус:

git status

Git должен сообщить вам, что был добавлен файл.

Затем добавьте (add) только что созданный файл, чтобы он стал частью изменений, которые вы зафиксируете (commit) (или сохраните) с помощью Git:

git add readme.txt

Наконец, зафиксируйте (commit) эти изменения в истории репозитория с коротким (m) сообщением с описанием обновлений.

git commit -m "Created readme"

Шаг: внесите больше изменений

Теперь добавьте еще одну строку в readme.txt и снова сохраните файл.

В терминале вы можете просмотреть разницу (diff) между файлом сейчас и тем, каким он был при последнем коммите.

Попросите Git, чтобы он показал вам разницу (diff) :

git diff

Теперь, используя то, что вы только что узнали выше, зафиксируйте это последнее изменение.
