Tạo một file trong repository mới, thêm vài thứ trong đó và "commit" sự thay đổi đó với Git.

Commits

Những commit ("cam kết") là cốt lõi của việc sử dụng Git. Đó là những thời điểm mà bạn lưu và mô tả việc mà bạn đã hoàn thành, nó đánh dấu cho lịch sử các thay đổi trên timeline của project.

Tạo một File mới

Giờ bạn đã có repository, hãy add thêm file vào đó.

Mở text editor và tạo ra một file trống. Hãy viết vài dòng, kiểu như "Xin chào, Việt Nam!", và lưu file dưới tên 'readme.txt'trong folder 'hello-world' bạn vừa tạo trong challenge trước.

Kiểm tra trạng thái và thêm commit thay đổi

Tiếp theo, kiểm tra status (trạng thái) của repository để thấy những thay đổi, nếu có. Bạn biết mình đã thay đổi thứ gì đó, vậy Git thì sao?

Hãy đảm bảo bạn vẫn đang làm việc trong directory 'hello-world' của mình trong lúc bạn chạy những dòng lệnh. Sử dụng Git để xem sự thay đổi trong repos của bạn:

Đầu tiên, kiểm tra trạng thái:

git status

Git sẽ cho bạn biết có một file đã được thêm mới.

Sau đó add file bạn vừa tạo để nó trở thành một phần trong thay đổi mà bạn sẽ commit (hay lưu lại) với Git:

git add readme.txt

Cuối cùng, commit tất cả những thay đổi đó vào lịch sử của repos với một thông điệp ngắn (m - message) để mô tả việc cập nhật.

git commit -m "Created readme"

Hãy tiếp tục tạo thêm thay đổi

Giờ hãy thêm vào những dòng khác trong file 'readme.txt' và lưu nó lại.

Trong terminal, bạn có thể xem được sự khác biệt giữa file hiện tại và chính nó trong commit trước đó.

Hãy nói với Git để thấy sự khác nhau (difference) diff:

git diff

Và bây giờ, với những gì bạn đã học ở trên, nhớ commit những thay đổi cuối nhé!.
