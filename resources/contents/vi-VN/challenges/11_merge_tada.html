Ghép nhánh cục bộ, xóa nhánh và kéo nhánh từ "thượng nguồn".

Làm sạch

Khi bạn không cần đến các nhánh nữa, bạn có thể xóa cả ở repos cục bộ lẫn remote repos. Việc này hữu ích với việc bạn không phải kết thúc với một nhánh cũ không cần đến.

Merge Locally - Ghép nhánh cục bộ

Pull request của bạn đã được ghép! Nhánh của bạn được ghép vào nhánh 'gh-pages' của bản gốc trên GitHub. Bạn cũng có thể ghép nhánh cục bộ.

Đầu tiên, chuyển đến nhánh mà bạn muốn ghép vào, trong trường hợp này là nhánh 'gh-pages'.

git checkout gh-pages

Giờ thì nói cho Git nhánh mà bạn muốn merge, chức năng với nhánh sẽ bắt đầu với "add-".

git merge <BRANCHNAME>

Làm cho mọi thứ trở nên gọn gàng với việc xóa nhánh chức năng. Nhánh giờ đã được ghép, bạn không cần đến nhánh này nữa:

git branch -d <BRANCHNAME>

Bạn cũng có thể xóa nhánh từ remote của GitHub:

git push <REMOTENAME> --delete <BRANCHNAME>

Pull về từ thượng nguồn (Upstream)

Cuối cùng nhưng không kém phần quan trọng, bản gốc đã thay đổi vì pull request của bạn đã được ghép vào—Reporobot thêm tên bạn vào website! Nếu bạn pull những thay đổi này từ upstream bạn sẽ cập nhật và nó sẽ có phên bản đi kèm. Bạn có thể xem tại: yourusername.github.io/patchwork.

Để pull từ upstream gốc:

git pull upstream gh-pages

Bạn đã tạo một repos cục bộ, một remote repos, cộng tác, gửi lên, kéo về và tham gia phát triển cho cộng đồng mã nguồn mở với hàng triệu nhà phát triển khác!
