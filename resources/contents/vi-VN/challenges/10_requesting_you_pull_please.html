Pull Requests

Thông thường, khi bạn thay đổi và cải tiến cho project mà bạn fork về, bạn sẽ muốn gửi tất cả những thay đổi ấy đến người bảo trì cho bản gốc và request (yêu cầu) họ pull (kéo về) những thay đổi này vào bản gốc để mọi người có thể trải nghiệm lợi ích từ bản cập nhật - đó gọi là pull request.

Chúng tôi muốn thêm bạn vào một danh sách những người đã hoàn thành workshop, hãy tạo một pull request thêm vào đó là username gửi đến bản gốc: github.com/jlord/patchwork.

Tạo một pull request

Vào repository gốc mà bạn fork về GitHub, trong trường hợp này làhttp://github.com/jlord/patchwork.

Thường thì GitHub sẽ nhận dạng nếu bạn đã đẩy một nhánh lên bản fork và hiển thị nó ở đầu trang của bản gốc. Nếu bạn thấy nhánh cùng với tên branch bạn đã tạo, bạn có thể click vào để tạo một Pull Request. Nếu không:

  • Nhấp chuột vào nút màu xanh 'New pull request'.
  • Chọn nhánh cùng với các thay đổi mà bạn muốn gửi lên. Nó sẽ trả về duy nhất cùng với tên nhánh của bạn.

Giờ bạn sẽ thấy một trang kèm theo chi tiết của pull request mà bạn đang gửi lên. Trang này hiển thị ra tất cả các bản commit và thay đổi, sự khác nhau của những file cùng dạng, kết nối với pull request của bạn để so sánh với nhánh 'gh-pages' ở nhánh gốc.

Nếu repos gốc có một tài liệu cộng tác - contribution documentation, GitHub sẽ liên kết tới nó. Đây là tài liệu từ người sở hữu repos, mang những thông tin làm thế nào để có thể cộng tác tốt nhất, rất hữu ích khi bạn đọc được nếu bạn nhận cộng tác!

Nếu mọi thứ trên trang trông có vẻ tốt như bạn mong đợi:

  • Click 'Create pull request'
  • Thêm vào mô tả cho thay đổi mà bạn đề xuất tác giả gốc pull về.
  • Click 'Send pull request'!

Bingo! Bạn đã gửi một pull request—sẽ mất vài giây để trình lên.

Nếu tất cả hoạt động tốt với pull request của bạn, nó sẽ được ghép vào nhánh cùng lúc. Nếu nó không được tự động ghép trong vài phút, có vẻ như bạn có vài bình luận từ @reporobot với lý do của việc bạn không được ghép nhánh. Nếu gặp trường hợp này, hãy đóng pull request trên GitHub, thay đổi những điều cần thiết trên nhánh của bạn, đẩy tất cả các thay đổi lên bản fork và mở lại pull requestm việt này sẽ gây chú ý để @reporobot xem lại.
